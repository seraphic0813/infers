# ナローフォーカス エントリー手法(正)

> **本書はエントリー判定ロジックの唯一の正である。** エントリーに関して本書と
> [phase1-architecture.md](phase1-architecture.md) / [../CLAUDE.md](../CLAUDE.md) が
> 矛盾する場合、**本書を優先**する。防御・執行・出口(SL/建値SL/半分利確/ランナー決済)は
> 引き続き設計書 §6 を正とする。

## 0. 設計思想

上位足(マクロ)のトレンドと下位足(ミクロ)のタイミングが **完全に同調した瞬間のみ** を
狙う、厳格なマルチタイムフレーム(MTF)戦略。エントリー執行までに **3つのゲートを順番に**
クリアする。各ゲートは AND 条件であり、1つでも不成立なら **NO-TRADE**(防御は LLM 非依存)。

各テクニカル要素の役割(原典 Narrow Focus 手法に準拠。2026-06-13 訂正):

| 要素 | 役割 |
|---|---|
| **ダウ理論** | **トレンド判定の主役**(高値・安値の更新でトレンド把握、基本原則④=高値切り下げ/安値切り上げで転換判定)。マクロ環境認識の第一歩 |
| **エリオット波動** | トレンドの構造的読み(第何波か)。**第2波の押し目を狙う文脈** |
| **200SMA** | **マクロ方向の確認 + ミクロのグランビル・トリガー**(90SMA と対等)。D1/H1/M5 で使う |
| **90SMA** | ミクロの動的サポレジ(グランビル) |
| **水平レジサポ** | 意識された水平ラインへのタッチ(コンフルエンス要素) |
| **フィボナッチ** | 押し目ゾーンの特定(38.2/50/61.8/78.6%)と **利確目標(161.8%)**。**コンフルエンスの対等な加点要素にはしない** |
| **RSI** | **マルチTF(M5/H1/D1)のコンフルエンス**。各足の極値(≤30買/≥70売)が根拠 |

## ゲート1: マクロ環境認識(トレンド・シールド)

相場の支配的方向を **ダウ理論(高値・安値の更新; 基本原則④で転換判定)** で確定する
(マクロTF: D1/H1)。確定した方向と**逆のエントリーは 100% 遮断**(NO-TRADE)。

- **200SMA位置**(価格が 200SMA の上=買い環境 / 下=売り環境)で方向を**確認補強**する。
- **エリオット波動**で「現在第何波か」を読み、**第2波の押し目を狙う文脈**を確定する。

> **訂正(2026-06-13)**: 旧版は「200SMA位置で方向を決定」「ダウは構造文脈」としていたが、
> 原典では **トレンド判定の主役はダウ理論**。200SMA は方向確認 + ミクロのグランビル・トリガーに
> 役割を戻した。これは原典・旧設計書・現実装(ダウベース)と整合する。

## ゲート2: コンフルエンス(独立根拠の重複)

ゲート1で方向確定後、押し目/戻りゾーンで以下を評価し、**規定数(2〜3件)以上の重複** を
エントリーの絶対条件とする。

### 評価する独立根拠
1. **ダウ理論の順行**: トレンド方向に高値・安値を更新中(押し目から再進行する局面)
2. **200SMAグランビル**: 200SMA への反発(**マクロ/ミクロ問わず**。90SMA と対等に扱う)
3. **90SMAグランビル(第2波)**: 90SMA 付近への綺麗な押し目(買い)/戻り(売り)
4. **水平レジサポ**: 意識された水平ライン(確定スイング群から生成)へのタッチ
5. **RSIトリガー(マルチTF)**: **M5/H1/D1** いずれかの RSI が ≤30(買)/≥70(売)に到達、
   またはそこから反発/反落(複数TFで重なるほど強い)
6. **40%深さスクリーニング**: 後述(原典に無い、システム化のための追加フィルター)

### 40%深さスクリーニング(確定仕様)

基準スパン = **直近スイング(直近高値〜直近安値の値幅)**。

```
ロング:   (現在値 − 直近安値) / (直近高値 − 直近安値) ≤ 0.40   → 下方40%(安値に近い深い押し目)
ショート: (直近高値 − 現在値) / (直近高値 − 直近安値) ≤ 0.40   → 上方40%(高値に近い深い戻り)
```

**採用解釈: 深い押し目**(安値/高値に近い側)。

> **原典との差異**: ソース資料に「下方40%(0.40)」という具体的数値ルールは**存在しない**。
> これは、エリオット第2波の絶対原則(第1波の始点を割らない=ギリギリまで引き付ける)と、
> バックテスト検証から導いた **システム化のための追加フィルター**。「負けないトレード」思想を
> システム言語に翻訳したルールであり、自動売買での適用は妥当だが、原典の人間的裁量より厳格。

**理由**: エリオット第2波は、第1波(下落トレンドを覆す急騰)の後、市場の大半が半信半疑で
売ってくるため、フィボ 61.8〜78.6% の **全戻しに近い深い押し目** になるのがセオリー。
直近安値の **すぐ外側に SL を置ける** ため、損切り幅を極小化し R/R を最大化する
Narrow Focus 固有の引き付けルール。

> **実装上の前提(重要)**: この「深い押し目で勝てる」性質は、**本物のエリオット第2波** で
> あることが前提。マクロシールド(ゲート1)と正しい第2波判定が無いと、深い押し目は単なる
> トレンド転換(落ちるナイフ)になり負ける(過去実装の実測で確認済み: 深さ0.77 は負け先行、
> 浅い0.60 が勝ち = それらは第3波内部/第5波の局面だった)。したがって 40%深さスクリーニングは
> **ゲート1とエリオット文脈と必ずセット** で機能する。単独で有効性を測ってはならない。

## ゲート3: ファンダメンタルズ遮断(ニュース・インターラプト)

ゲート1・2のテクニカル条件をすべて満たしても、重要経済指標(米雇用統計 / CPI / FOMC 等)の
発表スケジュールを最終チェックする。**発表前後30分** などノイズが激しい時間帯は、全テクニカル
根拠を無視して **新規エントリーを全面禁止**(見送り)とする。

- 必要データ: 経済カレンダー(指標日時)。現状未保有のため、データ源の整備が前提。
- 設計書 §6.5 のスプレッド異常拒否は別物(スパイクの事後的な代理)であり、本ゲートを代替しない。

## 防御・出口(設計書 §6 を正とする)

損切り / 建値SL / 半分利確 / ランナー決済は 100% 決定論(LLM 非依存)。半分利確 = RSI 極値 or
重要SR到達、ランナー利確 = フィボ161.8% or ダウ転換。詳細は
[phase1-architecture.md](phase1-architecture.md) §6.4。
