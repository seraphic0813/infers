## 2. 発見した問題 (重要度順)

### P1 【プロンプトバグ・最重要】ambiguity の意味論がコードと真逆

- コードの定義 ([elliot.py](../src/infers/analysis/elliot.py#L99-L102)):
  ambiguity = 波カウント1位と2位のスコア差。**小さい=候補拮抗=曖昧、
  大きい(〜1)=解釈一意**。1は「候補単一」のセンチネル (最も一意)。
- nf-v2 プロンプト §2/§3.3 の記述: 「**小さいほど解釈が一意=信頼度が高い**」 — **真逆**。
- 影響 (実測): 最も解釈が一意な群 (amb≥0.3、7,297件=41%) の **74.8%がNO_GO**。
  NO_GO理由の圧倒的1位が「ambiguity=Xが高く波カウント拮抗」(誤った教義への忠実な適用)。
  GO 33件は最も曖昧な群 (amb<0.15) に偏った。
- ただし反実仮想では ambiguity はどちらの向きにも予測力なし
  (バケット別 avgR は -0.10〜+0.03 でノイズ範囲)。
  → **修正しても勝率は上がらないが、不当な NO_GO 数千件が解放されエントリー数が大幅に増える**。
  同時に、予測力がない以上 GO/NO_GO を分ける主要因子から外すべき。

### P2 【構造】L1 (Haiku) の GO に選別能力がない — 逆指標ですらある

| 群 | 反実仮想 (固定SL vs fib_target先着) | 実トレード |
|---|---|---|
| L1_ONLY の GO (14件) | 約定5 → **0勝5敗**。expiry∞でも **0勝14敗** | 5約定 **全敗** |
| L1:WAIT (5,911件) | 勝率22.2% avgR-0.013 | - |
| L1:NO_GO (11,868件) | 勝率22.5% avgR-0.049 | - |
| **L2:GO (10件)** | 約定2 → **2勝 (avgR+4.30)** | 2約定: BE微損 + **大勝ランナー** |
| **L2:WAIT (9件)** | expiry×6で全約定 → **0勝9敗** (負けを正しく回避) | - |

- L1 の GO/WAIT/NO_GO 間に成績差がない = L1 は選別できていない。
  L1 の GO 33件中7件は §5 の RSI 要件を満たさないのに GO (基準の一貫適用に失敗)。
- L2 (Fable 5) はサンプル極小ながら一貫して正しい方向
  (GO は勝ち、WAIT は反実仮想で全敗の群)。判定理由の質も高い
  (risk=1tick の構造的脆弱性の指摘など)。
- → **GO の裁量は L2 に集約し、L1 は「明確な異常・基準違反の除外フィルタ」に縮小**
  (または廃止して L0 決定論フィルタ + L2 のみの2段に)。
  L1 への $30 は選別能力ゼロに費やされた。

### P3 【執行】約定率29%の原因は「expiry 60分 × 深い指値」— ただし無差別延長は逆効果

- 失効17件は全て expired (invalidated ゼロ)。eta_bars=1-12 → expiry は一律60分。
- expiry 感度分析: 延長すると約定率は 64%→99% に上がるが、
  **L2:GO の質は ×1 が最高 (+4.30R) で、延長するほど劣化** (×3: +1.12R、∞: +0.77R)。
  「eta_bars 内に到達してこそ合流点」という設計思想をデータが支持。
- → expiry はむしろ維持し、**失効後の再発行** (毎確定足で合流点を再計算して
  新プランとして再提案) で機会を増やすのが正しい。現状は cooldown_bars=12 が
  再発行を阻害している (失効検出時に cooldown を解除する等)。

### P4 【プラン品質】rsi_band の82%が退化 (lo=hi の点)

- [propose_limit_orders](../src/infers/analysis/future_discretion.py#L321) の
  best セル選択が同スコアで k 最小を選好 → k=1 ではRSI予測4パスが全て同一になり
  バンドが点に退化。82% (14,633件) が該当。
- 退化群の GO 率 0.10% vs 非退化 0.57%。プロンプト§3.1 の
  「certain/possible」区別が機能不全。
- → 修正: rsi_band をグループ内全セルの (min lo .. max hi) で合成する
  (provider.py の1行変更)、または best 選択タイブレークを k 中央へ。

### P5 【プラン品質】eta_bars の98.7%が「1-12」(全域) で情報価値ゼロ

- 価格方向のセル連結で k レンジがほぼ常に全域化。
- プロンプト§3.5「窓が広い→減点」がほぼ全件に機械的に発動し、WAIT 量産の一因
  (WAIT理由の上位常連)。
- → グループの k 範囲を best セル近傍に制限する、または §3.5 の重みを下げる。

### P6 【コスト】プランの50.5% (9,002件) が families=SR,FIB のみで、その94%がNO_GO

- 中核根拠 (SMA/RSI) 欠如はプロンプトが明示的に減点 → L1 費用の約半分が
  「ほぼ自動NO_GO」の判定に消えた。
- → L0 で「families に SMA または RSI を含む」を要求してから LLM へ
  (EscalationPolicy 拡張 or provider 側フィルタ)。判定費用半減。

### P7 【執行・要設計判断】追撃後の「建値SL」がポジション全体では大損になる

- 2025-06-10: ADD (打診+約600tick上で約定) 後に建値SL移動 → SLヒットで **-1,192**
  ($-11.92)。SL は「打診玉の建値+2tick」([sm.py:239](../src/infers/execution/sm.py#L239))
  のため、追撃玉には -600tick の損切り。
- これが be_sl_exit_rate=0% の原因 (全体PnLが負なので建値退出と判定されない)。
- → 設計判断が必要: 「建値」を (a) 打診建値のまま (現状)、
  (b) 追撃込みの平均建値、(c) 追撃時にSLを平均建値+offsetへ再計算、のどれにするか。
  (b)/(c) なら SL単調性を保ったまま実装可能 (追撃は利益方向でのみ発生するため)。

### P8 【手法の本質】エントリー選別にエッジはなく、エッジの源泉はランナー

- 全18,070プランの簡易執行 (固定SL+fib_target利確) は avgR -0.04〜+0.00 —
  **どの特徴量で切ってもエントリー時点の優位性は検出されない**
  (cluster_score にも単調性なし)。
- 実トレードの利益は全てランナー1本 (+$2,328、2022-12エントリーで3年半保有) に依存。
- 含意: **「エントリーの質」を追い込むより「数を確保して、BE防御+ランナーの
  非対称性で損小利大を作る」のが正しい方向**。勝率22%・avgR≈0 のベースラインでも、
  ランナーが乗れば期待値は正になる。ただし P7 の修正が前提。

## 4. 推奨アクション (優先度順)

### Tier 1 — バグ修正 (再ジャッジの前提。すべて無料のコード/プロンプト修正)
1. **プロンプト ambiguity の向きを修正** (P1)。同時に「amb=1 は候補単一の
   センチネル」を明記し、判定における重みを下げる (予測力なしのため)。
2. **WAIT の廃止または意味変更**: verdict はキャッシュで恒久化されるため、
   「WAIT=後で再評価」は成立せず実質 NO_GO。現プロンプトの「迷ったらWAIT」は
   「迷ったら永久に捨てる」になっている。GO/NO_GO の2値 + confidence に寄せる。
3. **rsi_band のグループ合成** (P4) と **eta_bars の意味回復** (P5)。
4. **判定基準の再設計**: §5 の AND 条件 (RSI∧RR∧amb∧cs) を撤廃し、
   「シナリオ無効化リスク (risk=1tick等の構造チェック) + RR + RSI整合」の
   加点方式へ。cluster_score/ambiguity は参考情報に格下げ (予測力なしのため)。

### Tier 2 — 構造変更 (要実装)
5. **L1 の役割縮小**: L0 決定論フィルタ (SMA/RSI必須・RR≥1.5・risk≥最小幅) →
   通過分を直接 L2 へ。L1 は廃止候補 (選別能力ゼロが実測されたため)。
6. **失効時再発行** (P3): cooldown と失効の連動。
7. **BE-SL の建値定義** (P7): 上記3案からの設計判断 — これは手法のオーナー判断事項。

### Tier 3 — 検証プロセス
8. nf-v3 パイロット (数百件・$1-3) → 判定分布確認 → 全件 → リプレイ。
9. **ウォークフォワード分割**: 本分析でこの5年データに合わせた修正を入れた時点で、
   同一データの再評価は楽観バイアスを持つ。修正の最終評価は
   未使用期間 (新規データ取得 or BTCUSD) で行うこと (設計書 §8)。
10. BTCUSD 追加でサンプル倍増。

## 5. 今回の分析手法の限界 (正直な注記)

- 反実仮想は「固定SL vs fib_target 先着」の簡易執行であり、BE移動・半分利確・
  追撃・ランナーを再現していない。**群間の相対比較**には有効だが、
  絶対期待値は実機 (FSM) と異なる。
- L2 の優位性は n=19 での観測。確証には nf-v3 後の追加サンプルが必要。
- 同一バー内の到達順序 (SL と target が同一バーで両方タッチ) は SL 優先の
  保守側で解決しており、勝率はやや過小評価されている可能性がある。
